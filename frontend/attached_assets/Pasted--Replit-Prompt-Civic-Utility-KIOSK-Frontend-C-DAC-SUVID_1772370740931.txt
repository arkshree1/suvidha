# Replit Prompt: Civic Utility KIOSK Frontend (C-DAC SUVIDHA Challenge)

---

## PROMPT TO PASTE INTO REPLIT:

---

Build a **complete React.js frontend** for a **Unified Civic Services Self-Service KIOSK** for the C-DAC SUVIDHA Challenge. This is a touch-screen kiosk interface used in Indian civic utility offices — Electricity, Gas, and Water/Municipal. No backend is needed — use mock/static data throughout.

---

### Tech Stack
- **Frontend:** React.js with React Router v6
- **Styling:** Plain CSS and CSS Modules only — NO Tailwind, NO Bootstrap, NO external CSS frameworks
- **Icons:** Lucide React
- **State:** React useState and useContext only
- **Backend:** Not required — use mock JSON data and hardcoded state

Write all styles in `.module.css` files or a `styles/` folder with regular `.css` files. Use CSS variables for the color palette and spacing. Do not import any CSS utility library.

---

### Design System — UX4G (Government of India Guidelines)

**CSS Variables (define in `index.css` or `variables.css`):**
```css
:root {
  --color-primary: #006EB3;
  --color-secondary: #F26522;
  --color-bg: #F5F7FA;
  --color-surface: #FFFFFF;
  --color-success: #28A745;
  --color-warning: #FFC107;
  --color-error: #DC3545;
  --color-text: #212529;
  --color-text-muted: #6C757D;
  --color-border: #DEE2E6;
  --radius-card: 16px;
  --radius-btn: 12px;
  --shadow-card: 0 4px 16px rgba(0,0,0,0.10);
  --shadow-btn: 0 2px 8px rgba(0,0,0,0.12);
}
```

**Typography:**
- Import `Noto Sans` and `Noto Sans Devanagari` from Google Fonts (supports Hindi script)
- Headings: 32–48px, font-weight 700
- Body text: 20–24px (readable from 60cm — standard kiosk viewing distance)
- Button labels: 22–26px, font-weight 600

**Touch Targets:**
- Every button and tappable element: minimum 80×80px
- Button minimum height: 72px
- Gap between interactive elements: minimum 16px
- No styles that only apply on hover — design for touch, not mouse

**Icons:** All icons must be at least 32px and appear alongside text labels (never icon-only)

**Accessibility:**
- WCAG 2.1 AA colour contrast on all text
- Visible focus ring on all interactive elements (use `outline` in CSS, do not use `outline: none`)
- `aria-label` on every button and input
- High-contrast mode toggle (adds a `.high-contrast` class to `<body>`, override variables inside it)
- Font size controls: +A and -A buttons always visible in header, adjusting a `--base-font-size` variable

---

### Multilingual Support

Add a language switcher in the header:
1. English (default)
2. Hindi (हिंदी)
3. Marathi (मराठी) — placeholder translations acceptable

Store the selected language in a `LanguageContext`. All screen labels, headings, and button text must come from a `translations.js` file — no hardcoded English strings in components.

---

### Screen Layout — Portrait Kiosk (1080×1920px)

Every screen shares the same shell:
- **Header** (fixed top): Government of India emblem + "SUVIDHA KIOSK" name, live time and date, language switcher (EN | हिं | मर), high-contrast toggle, +A / -A font size buttons, Home button
- **Announcement ticker** (just below header): horizontally scrolling text for notices, e.g. "Power outage in Ward 5 on 15 Dec from 10am–2pm"
- **Main area** (scrollable): page content
- **Footer** (fixed bottom): "Need Help? Call 1800-XXX-XXXX", session countdown timer showing "Session expires in X:XX"

### Welcome Screen (Home / Auth)

- Large greeting: "नमस्ते / Welcome to SUVIDHA KIOSK"
- Below it: "Tap a card below to continue"
- Two large login option cards side by side:
  1. Mobile OTP — citizen enters 10-digit mobile number on the on-screen numpad, then enters any 6-digit OTP (all OTPs accepted in mock)
  2. Aadhaar — citizen enters 12-digit Aadhaar number, mock verification passes automatically
- A smaller "Guest / Walk-in" text button below for limited access
- Language selection is prominent on this screen
- Auto-logout after 120 seconds of no interaction; show a warning modal at 30 seconds remaining

### Session Management

- Track last interaction time using a global event listener on `window` (touch, click, keydown)
- Auto-logout after 120 seconds of no interaction
- At 30 seconds remaining, show a full-screen warning modal: "Your session will end in 30 seconds. Tap anywhere to continue."
- On logout: reset all context state, navigate to `/`
- Session countdown always visible in the footer

---

### On-Screen Input Components

These are required — the kiosk must work without a physical keyboard:

**NumericKeypad component:**
- Buttons for 0–9, backspace, clear
- Large square buttons (minimum 80×80px each)
- Used for mobile number, Aadhaar, consumer numbers, OTP, meter readings

**VirtualKeyboard component:**
- Full QWERTY layout
- Shift for uppercase, backspace, space, clear
- Large keys (minimum 60px height)
- Used for name fields, address, complaint description

Both components take an `onChange` callback and a `value` prop.

### CSS Animations (define in `animations.css`, no libraries)

- `fadeIn`: opacity 0 → 1, used on every page mount
- `slideUp`: translateY(20px) → 0, used on cards loading in
- `pulse`: scale 1 → 1.05 → 1, looping, used on "Tap to Start" on the welcome screen
- `ripple`: expanding circle on button press, triggered by adding a `.ripple` class via JS
- `ticker`: continuous horizontal scroll for the announcement banner
